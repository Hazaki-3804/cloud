Tuần 3: Bảo mật Web - HTTPS, HTTP Headers & OWASP

Khám phá các khái niệm cơ bản về bảo mật web và các lỗ hổng phổ biến

1. HTTP, HTTPS và TLS

1.1 HTTP là gì?

HTTP (HyperText Transfer Protocol) là giao thức dùng để truyền dữ liệu giữa Trình duyệt và Máy chủ web.

Nhược điểm:

  • Dữ liệu truyền ở dạng plaintext
  • Dễ bị nghe lén, sửa đổi, giả mạo (Man-in-the-Middle)

Minh họa kết nối HTTP không an toàn

Quy trình bắt tay TLS

1.2 HTTPS và TLS

HTTPS (HTTP Secure) sử dụng TLS để đảm bảo:

  1. Bí mật (Confidentiality): Dữ liệu được mã hóa.
  2. Toàn vẹn (Integrity): Dữ liệu không bị thay đổi.
  3. Xác thực (Authentication): Xác minh danh tính máy chủ.

2. HTTP Security Headers

HTTP Security Headers giúp kiểm soát cách trình duyệt xử lý nội dung để ngăn chặn tấn công XSS, Clickjacking,...

HeaderChức năng
Strict-Transport-SecurityBắt buộc dùng HTTPS
Content-Security-PolicyNgăn chặn tấn công XSS
X-Frame-OptionsChống Clickjacking
X-Content-Type-OptionsNgăn MIME sniffing

3. OWASP Top 10 (2021-2025)

OWASP là tổ chức mở chuyên về cải thiện bảo mật phần mềm. OWASP Top 10 liệt kê các lỗ hổng nguy hiểm nhất hiện nay.

  • Nâng cao nhận thức an toàn web
  • Làm tài liệu chuẩn cho kiểm thử bảo mật

OWASP - Tổ chức bảo mật hàng đầu

Nhóm rủi roMô tả ngắn
Broken Access ControlNgười dùng vượt quyền truy cập.
Security MisconfigurationLỗi cấu hình server/app.
InjectionNhập dữ liệu độc hại vào hệ thống.
Cryptographic FailuresSai lệch mã hóa dữ liệu.

4. OWASP Juice Shop

Juice Shop là một ứng dụng web không an toàn dùng để thực hành bảo mật:

  1. Giúp sinh viên hiểu rõ bản chất lỗ hổng.
  2. Thực hành pentest trong môi trường an toàn.
  3. Liên hệ lý thuyết với thực tế.

5. Phân tích rủi ro & Cloud

Tại sao Broken Access Control nguy hiểm hơn trên Cloud?

Trên Cloud, tài nguyên công khai Internet và bề mặt tấn công rộng lớn hơn. Nếu lỗi phân quyền xảy ra, kẻ tấn công có thể khai thác từ bất cứ đâu mà không cần vào mạng nội bộ.

Biện pháp bảo vệ:

  • Quản lý danh tính & quyền truy cập (IAM).
  • Áp dụng mô hình Zero Trust.
  • Sử dụng Web Application Firewall (WAF).
