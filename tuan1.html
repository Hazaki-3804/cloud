LAB TUẦN 1: AN TOÀN ĐIỆN TOÁN ĐÁM MÂY

Khám phá các khái niệm cơ bản, kiến trúc và mô hình dịch vụ của điện toán đám mây

Cloud Computing là gì?

Cloud Computing (Điện toán đám mây) là mô hình cung cấp tài nguyên CNTT (máy chủ, lưu trữ, cơ sở dữ liệu, mạng, phần mềm,…) thông qua Internet, thay vì phải cài đặt và quản lý trực tiếp trên máy tính cá nhân hay máy chủ vật lý.

Ví dụ phổ biến: Google Drive, AWS, Microsoft Azure, Cloudflare,…

Kiến trúc Cloud Computing cơ bản

Frontend (Client)

  • Trình duyệt web
  • Ứng dụng di động
  • Phần mềm client

Backend (Cloud Provider)

  • Máy chủ (Server)
  • Lưu trữ (Storage)
  • Cơ sở dữ liệu

Network (Internet)

Kết nối không dây hoặc có dây đóng vai trò cầu nối dữ liệu giữa người dùng và hệ thống cloud.

Các mô hình dịch vụ Cloud

IaaS

Cung cấp hạ tầng: máy ảo, mạng, ổ cứng. Người dùng tự cài hệ điều hành.

Ví dụ: AWS EC2, GCE

PaaS

Cung cấp nền tảng lập trình. Không cần quản lý hệ điều hành hay server.

Ví dụ: Heroku, Vercel

SaaS

Phần mềm hoàn chỉnh. Người dùng chỉ sử dụng trực tiếp qua web/app.

Ví dụ: Gmail, Microsoft 365

Website tĩnh thuộc mô hình nào?

Website tĩnh thường thuộc: PaaS hoặc SaaS. Đặc điểm: Chỉ gồm HTML/CSS/JS, không xử lý logic server.

  • Ví dụ: Cloudflare Pages, GitHub Pages, Netlify.

1

3
Phân phối toàn cầu

• Lưu trữ an toàn trên hạ tầng Cloudflare.

Mỗi lần cập nhật code → hệ thống tự động deploy lại ngay lập tức.

CDN – Content Delivery Network

CDN là gì?

  • Phân phối nội dung nhanh hơn đến người dùng.
  • Giảm tải đáng kể cho server gốc.

Quy trình phân phối

1

Xác định vị trí người dùng qua IP.

2

Trả dữ liệu từ máy chủ (Edge Server) gần nhất.

3

Lưu Cache để phục vụ cho các lần truy cập sau.

Vai trò của CDN

Về hiệu năng

  • Giảm độ trễ (latency).
  • Tối ưu hóa băng thông.

Về an toàn

  • Chống tấn công DDoS.
  • Tích hợp sẵn WAF và SSL/TLS.

Rủi ro bảo mật tồn tại

  • Lộ mã API Key / Token bí mật.
  • Cấu hình sai quyền truy cập tài nguyên.
  • Các lỗ hổng web: XSS, CSRF.
  • Phụ thuộc nhà cung cấp (Vendor Lock-in).
  • Tấn công vào tài khoản quản trị viên.

Giải pháp bảo mật đề xuất

Về kỹ thuật

  • Sử dụng HTTPS và bật WAF.
  • Nguyên tắc quyền hạn tối thiểu (Least Privilege).
  • Bật xác thực đa yếu tố (MFA).

Về quản trị

  • Kiểm tra log và sao lưu định kỳ.
  • Đào tạo nhận thức an toàn thông tin.
  • Kiểm tra bảo mật (Pentest) định kỳ.

CÁC BƯỚC TRIỂN KHAI WEBSITE TĨNH

Hướng dẫn chi tiết quy trình đưa website lên Cloudflare Pages thông qua GitHub.

1
Chuẩn bị tài khoản & mã nguồn

Tạo tài khoản GitHub và Cloudflare. Hoàn thiện bộ mã nguồn tĩnh (index.html, style.css, script.js).

2
Tạo Repository trên GitHub

Đăng nhập GitHub, tạo một kho lưu trữ (Repository) mới ở chế độ Public và tải toàn bộ mã nguồn lên đó.

3
Kết nối với Cloudflare Pages

Trong bảng điều khiển Cloudflare, chọn mục Pages -> Create a project -> Connect to Git.

4
Cấu hình & Triển khai

• Chọn đúng Repository vừa tạo.

• Framework preset: Chọn None.

• Nhấn Save and Deploy.

5
Kiểm tra & Cập nhật

Truy cập vào tên miền *.pages.dev được cung cấp. Kể từ nay, mỗi khi bạn Commit code mới lên GitHub, Cloudflare sẽ tự động cập nhật website.
